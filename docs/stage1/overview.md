# Ступень 1 — Период знакомства (День 0–4)

## Проблема

В момент регистрации у банка нет транзакционной истории клиента. Модели склонности не работают без данных. Классический онбординг отправляет всем один и тот же скрипт — конверсия низкая.

## Решение

Классификация по регистрационным данным: ОКВЭД, тип субъекта, канал привлечения, возраст бизнеса. На основе 50 признаков CatBoost определяет один из 8 бизнес-портретов и рекомендует якорный продукт.

## Состояние данных на Day 0

| Параметр | Статус |
|---|---|
| Регистрационные данные | ✅ Полные (ОКВЭД, регион, тип, канал) |
| Транзакционная история | ⚠️ 0–3 операции |
| Модели склонности | ❌ Нет — недостаточно данных |

## Pipeline

```
Клиент открыл счёт (День 0)
        │
        ▼
CatBoost: классификация по 50 признакам
        │
        ▼
Один из 8 портретов (P1–P8) + якорный продукт
        │
        ▼
Генерация sales-аргумента под тип взаимодействия
(banner / push-уведомление / голосовой скрипт)
        │
        ▼
Доставка клиенту через CRM / push / колл-центр
        │
        ▼
Сбор метрик взаимодействия (35 метрик по 5 уровням)
```
