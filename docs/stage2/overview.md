## Плановый подход

### Модели склонности

Для каждого якорного продукта — отдельная бинарная модель (CatBoost Binary):

```
model_propensity_acquiring.cbm     # Эквайринг
model_propensity_zpp.cbm           # Зарплатный проект
model_propensity_subscription.cbm  # Подписки
model_propensity_accounting.cbm    # Бухгалтерия
model_propensity_kopilka.cbm       # Налоговая копилка
```

### Ключевые поведенческие признаки

```
sum_deb_fl_30           # Сумма переводов физлицам за 30 дней
cnt_deb_atm_30          # Снятий наличных
abm_paymentfiz_rate_1m  # Доля платежей физлицам
balance_rur_amt_30_min  # Минимальный остаток
avg_check_30            # Средний чек
cnt_contractors_30      # Уникальных контрагентов
has_salary_pattern      # Признак регулярных выплат физлицам
```

### Генерация аргумента (Ступень 2)

```
SHAP-топ-3 признака для этого клиента
        │
        ▼
Интерпретация в текст:
  sum_deb_fl_30=450k  →  "Вы активно переводите физлицам"
  cnt_deb_atm_30=12   →  "Вы регулярно снимаете наличные"
        │
        ▼
LLM #1: персонализированный аргумент
        │
        ▼
LLM #2: compliance-проверка
        │
        ▼
Батч-генерация раз в сутки → CRM
```
