

Ты помогаешь строить обучающие данные для банковского онбординга МСБ (90 дней после открытия РКО).

## Задача 1 — классификатор приоритета сценариев (уже обучен)
Сегменты P1–P8 определяют порядок сценариев онбординга. Признаки клиента — из датасета онбординга (ОКВЭД, география, обороты, дни с ОГРН, категория бизнеса и т.д.).

## Задача 2 — склонность к продукту (текущая)
Для каждого продукта нужна оценка propensity_score ∈ [0,1] и 3–5 ключевых признаков для sales-аргумента (BRD).

Продукты пилота: zpp, alfa_payments, nachalo, trade_acquiring, internet_acquiring, tax_jar, savings, accounting

Ограничения онбординга:
- У новых клиентов мало данных первые ~30 дней — склонность менее уверенная.
- Якорные сервисы: мобильное приложение, бизнес-карта, кэшбэк, подписки переводов, счёт на оплату, налоговая копилка, накопительный счёт, бухгалтерия.
- Цель онбординга: ≥2 якорных сервиса.

## Связь задач
Клиенты для склонности — те же, что в обучении P1–P8. Сегмент P4 может слегка повышать релевантность отдельных продуктов, но не отменяет бизнес-логику BRD.

### Требуемый JSON (строго)
```json
{
  "propensity_score": <float 0..1>,
  "propensity_label": <0|1>,
  "top_features": [
    {"name": "<feature>", "value": <number|string>, "direction": "increases|decreases"}
  ],
  "segment_description_ru": "<1 предложение>",
  "sales_argument_draft_ru": "<3-4 предложения, без гарантий доходности, без давления>"
}
```

Согласуй score с сегментом и продуктом. Для ИП и «Начало» — выше склонность при нулевом пакете. Для ЗПП — выше при выплатах физлицам и молодых ЮЛ.
